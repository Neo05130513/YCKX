# 延成科讯管理系统正式交付验收清单

本文档用于项目交付、试运行和客户验收。验收目标是核心业务模块达到可培训、可试用、可回归验证的 95% 以上完成度；生产部署时仍需按实际环境接入数据库、对象存储、第三方设备平台和备份策略。

## 1. 验收范围

| 模块 | 验收负责人 | 完成标准 | 验收证据 |
| --- | --- | --- | --- |
| OA 审批 | 运营 / 管理层 | 审批台账、模板发起、节点流转、附件、催办、转办、批量通过、撤销删除、详情检查清单可用 | 后台 `/approvals` 页面操作截图、导出 CSV、审批流转记录 |
| 租赁订单 | 销售 / 资产 / 财务 | 创建订单、合同会签、合同归档、出库、收货、账单、回款、续租、退租、退租结清可闭环 | 后台 `/orders` 页面队列、订单详情、客户收货回写 |
| 系统管理 | 系统管理员 | 账号、角色、字典、接口、审批流、参数、审计日志、配置导出恢复、交付健康度检查可用 | `/settings` 交付健康度、系统配置 JSON、验收清单 CSV |
| 客户移动端 | 客户管理员 / 客户员工 | 首页待办、订单筛选、收货确认、电池查看、报修提交、账单权限、付款凭证入口可用 | 客户端 `/home`、`/orders`、`/receipts`、`/repair`、`/bills` |
| 帮助 / 操作文档 | 项目交付 | 内置帮助页和仓库文档覆盖模块操作、岗位路径、上线检查和回归步骤 | `/help` 页面、`docs/操作手册-核心流程.md` |

## 2. 模块验收项

### OA 审批

| 检查项 | 通过标准 |
| --- | --- |
| 审批工作台 | 可查看待我审批、超时/紧急、草稿待提交、附件待补队列 |
| 模板发起 | 可按业务模板自动带出字段、节点、风险项、建议附件 |
| 流程处理 | 支持提交、通过、驳回、转办、备注、撤销、删除 |
| 批量操作 | 支持批量催办、批量通过、批量转办、模板库查看 |
| 附件归档 | 新建和详情补充附件均能进入审批详情 |
| 审计追踪 | 详情可查看基础信息、表单字段、流程节点、流转记录、附件 |

### 租赁订单

| 检查项 | 通过标准 |
| --- | --- |
| 订单创建 | 可填写客户、联系人、租期、数量、月租、押金、账单日 |
| 合同流程 | 草稿发起会签，会签后可归档合同 |
| 出库绑定 | 可创建出库单并绑定 BT 码，订单进度同步 |
| 收货确认 | 后台和客户移动端均可确认收货，自动收货任务可执行 |
| 账单回款 | 可生成账单、确认账单、登记回款并同步欠款 |
| 续租退租 | 支持续租、退租、退租结清、完成订单和取消订单校验 |
| 履约队列 | 合同待归档、待出库、待收货、催收回款、退租结清队列可直接办理 |

### 系统管理

| 检查项 | 通过标准 |
| --- | --- |
| 账号管理 | 内部账号和客户账号可新增、编辑、启停、重置密码、删除 |
| 角色权限 | 支持内部后台和客户移动端分权，敏感权限有提示 |
| 字典维护 | 设备、资产、告警、报修、账单等状态字典可维护 |
| 接口配置 | 接口地址、鉴权、超时、负责人、启停和测试结果可维护 |
| 审批流配置 | 可新增、编辑、启停、删除流程，并从模板带入节点 |
| 系统参数 | 自动收货、逾期、阈值类参数可编辑并标注风险 |
| 交付检查 | 健康度覆盖账号、权限、接口、审批、审计、快照、参数风险 |

### 客户移动端

| 检查项 | 通过标准 |
| --- | --- |
| 首页待办 | 展示待收货、进行中报修、待付款账单或租赁订单 |
| 订单筛选 | 支持全部、待收货、租赁中、退租中、已完成 |
| 收货确认 | 展示出库时间、自动收货日、收货人、地址、BT 码并确认整单收货 |
| 报修入口 | 订单和电池页可进入报修，提交后后台可跟进 |
| 金额权限 | 客户管理员可看账单，客户员工不展示金额 |
| 付款凭证 | 欠款账单可提交付款凭证进入后台核验 |

## 3. 上线前检查

| 类别 | 必查项 |
| --- | --- |
| 环境变量 | API 地址、前端地址、数据库连接、文件存储、第三方平台密钥、管理员初始账号 |
| 数据库 | 生产库迁移完成，账号、角色、字典、审批流、系统参数初始化完成 |
| 文件存储 | 审批附件、报修图片、付款凭证、合同附件有持久化存储和访问权限 |
| 第三方接口 | 设备平台、定位、通知、短信或消息接口完成鉴权测试 |
| 账号权限 | 管理员、销售、资产、财务、售后、客户管理员、客户员工权限核对 |
| 备份回滚 | 数据库备份、配置快照、静态资源版本、回滚步骤确认 |
| 审计安全 | 关键操作日志、登录有效期、HTTPS、弱口令替换、敏感权限确认 |

## 4. 回归验证命令

```bash
pnpm typecheck
pnpm build
```

本地 mock 预览建议使用高位端口，并设置 `SKIP_PRISMA_CONNECT=true` 后再启动 API 和前端。验证结束后停止本地服务，避免 `data/mock-state.json` 持续写入运行噪声。
