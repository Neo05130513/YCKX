# 延成科讯管理系统 - 开发执行方案

## 1. 技术栈

| 层 | 技术 | 原因 |
|----|------|------|
| 后台前端 | Vue 3 + TypeScript + Vite + Element Plus | 企业管理系统成熟稳定，表格、表单、弹窗、权限菜单生态完善 |
| 客户移动端 | Vue 3 + TypeScript + Vite + Vant | 手机端网页，不做小程序，Vant 更适合移动端 |
| 后端 API | NestJS + TypeScript | 结构清晰，适合复杂业务模块、权限、审批流 |
| 数据库 | PostgreSQL | 适合订单、资产、财务、审批等关系型业务 |
| ORM | Prisma | 数据模型清晰，迁移方便 |
| 状态管理 | Pinia | Vue 官方推荐方案 |
| 图表 | ECharts | 工作台、财务、告警统计 |
| 鉴权 | JWT + RBAC | 支持内部角色、客户管理员、客户员工权限控制 |
| 包管理 | pnpm workspace | 适合 monorepo，多应用共享依赖和类型 |

## 6. 必须落地的业务规则

| 规则 | 实现方式 |
|------|----------|
| BT码唯一 | 数据库唯一约束 + 接口校验 |
| 出库前必须在库 | 出库接口校验电池状态 |
| 出库后状态变租赁 | 出库成功后事务更新 |
| 一个订单可多次出库 | 订单与出库单一对多 |
| 客户按出库单确认收货 | 客户端确认出库单 |
| 7天自动收货 | 后端任务或查询时自动判断 |
| 账单按月生成 | 根据订单账单日生成账单 |
| 账单7天后逾期 | 后端定时或查询时计算 |
| 客户员工看不到金额 | 前端隐藏 + 后端权限过滤 |
| 客户不能报修别人的电池 | 报修接口校验客户与 BT码关系 |
| 财务调整账单留痕 | Bill 调整记录 / OperationLog |

## 7. 执行顺序

1. 初始化 monorepo 项目。
2. 创建前后端基础工程。
3. 配置 TypeScript、基础样式变量。
4. 建立共享枚举和类型。
5. 建立 Prisma 数据模型。
6. 实现后端基础模块和 mock seed。
7. 实现后台登录和主布局。
8. 实现后台核心页面静态版。
9. 接入 mock API。
10. 实现客户移动端页面。
11. 实现核心业务规则接口。
12. 运行构建和基础检查。
13. 更新 README，说明如何启动项目。
