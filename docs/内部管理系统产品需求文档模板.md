## 8. 数据权限规则

内部系统如果不写数据权限，后面一定吵架。尤其是客户、金额、工资、合同这些敏感数据。

### 8.1 数据范围

| 数据类型 | 谁能看 | 谁能编辑 | 谁能删除 | 是否可导出 |
|----------|--------|----------|----------|------------|
| 员工信息 | HR / 管理层 | HR | 管理员 | 是 / 否 |
| 客户信息 | 所属销售 / 销售负责人 / 管理层 | 所属销售 | 管理员 | 是 / 否 |
| 合同信息 | 销售 / 财务 / 管理层 | 负责人 | 管理员 | 是 / 否 |
| 财务金额 | 财务 / 老板 / 授权人员 | 财务 | 管理员 | 是 / 否 |
| 项目信息 | 项目成员 / 管理层 | 项目负责人 | 管理员 | 是 / 否 |
| 审批记录 | 发起人 / 审批人 / 管理员 | 按流程 | 不建议删除 | 是 / 否 |

### 8.2 敏感操作

| 操作 | 是否需要权限 | 是否记录日志 | 是否二次确认 |
|------|--------------|--------------|--------------|
| 删除客户 | 是 | 是 | 是 |
| 导出客户 | 是 | 是 | 是 |
| 修改合同金额 | 是 | 是 | 是 |
| 删除审批记录 | 不建议允许 | 是 | 是 |
| 修改财务收款 | 是 | 是 | 是 |
| 重置员工密码 | 是 | 是 | 否 |

## 13. 风险与问题

### 13.1 已知风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 需求范围过大 | 周期拉长，开发失控 | 分期建设，先做 P0 |
