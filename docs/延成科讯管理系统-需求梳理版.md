# 延成科讯管理系统 - 需求梳理版

## 1. 项目定位

延成科讯管理系统是一套围绕“锂电池资产全生命周期管理”的内部业务平台，核心管理对象为带有唯一 BT 码的锂电池组。

系统需要覆盖电池设备数据汇总、告警管理、租赁订单、固定资产、售后报修、财务账单、业务审批等流程，并提供客户手机端网页用于客户查询订单、电池、账单及提交报修。

一句话定义：

> 以 BT 码电池组为核心，打通设备数据、资产流转、客户租赁、售后维修、财务账单和业务审批的综合管理系统。

## 2. 系统端划分

| 端 | 使用对象 | 主要用途 |
|----|----------|----------|
| 内部管理后台 | 公司内部员工 | 设备、告警、订单、资产、售后、财务、OA审批、系统配置 |
| 客户移动端网页 | 外部客户 | 查看订单、电池、账单、欠款、确认收货、提交报修 |

说明：

- 客户端不做微信小程序，改为手机端网页。
- 客户可将网页保存到手机桌面使用。
- 客户账号由公司后台开通。
- 一个客户公司可开多个账号。

## 6. 核心业务流程

### 6.1 租赁订单与出库流程

```text
销售创建租赁订单
→ 填写客户、租期、价格、账单日等信息
→ 上传或填写合同信息
→ 发起销售租赁合同会签
→ 审批通过后订单生效
→ 订单进入待出库状态
→ 资产管理部门创建出库单
→ 出库时选择在库状态的 BT码电池
→ 出库成功后电池绑定客户和订单
→ 电池状态变为租赁
→ 客户移动端确认收货
→ 超过 7 天未确认则系统自动确认收货
```

### 6.2 账单与回款流程

```text
订单配置账单日
→ 系统按月生成应收账单
→ 财务确认账单
→ 财务可手动调整账单金额并填写原因
→ 客户管理员查看每期账单和欠款总额
→ 财务登记回款
→ 系统更新已收、欠款、逾期状态
→ 账单生成后 7 天未结清则自动逾期
```

### 6.3 客户报修流程

```text
客户登录移动端网页
→ 扫描电池二维码或输入 BT码
→ 系统校验 BT码是否属于当前客户有效订单
→ 不属于则提示“该电池不属于您的订单”
→ 属于则填写故障描述、上传图片、填写地址
→ 客户点击提交
→ 后台生成报修单
→ 售后或技术人员受理处理
→ 必要时电池状态变为维修
→ 维修完成后重新入库
→ 如需继续给客户使用，再次出库
```

### 6.4 内部报修流程

```text
技术人员或资产人员登录后台
→ 扫描或输入 BT码
→ 填写故障描述、图片、位置/地址
→ 提交内部报修单
→ 技术/售后处理
→ 维修完成后进入待入库处理
→ 资产管理人员重新入库
```

## 7. 模块梳理

### 7.1 设备管理

设备管理负责汇总 8 个第三方平台的电池运行数据。

核心能力：

- 第三方平台接口配置。
- 定时主动拉取数据。
- 字段映射与数据清洗。
- 按 BT码统一展示电池数据。
- 展示来源平台、最后同步时间、设备运行状态。
- 识别接口异常、字段缺失、BT码冲突等问题。

注意：

- 第三方平台数据用于展示设备运行状态。
- 本系统资产状态由业务流程控制。
- 设备运行状态和资产业务状态不能混为一个字段。

### 7.2 告警管理

告警管理汇总来自设备数据的异常信息，仅内部可见。

核心能力：

- 告警列表。
- 告警等级。
- 告警类型。
- 来源平台。
- 关联 BT码。
- 关联客户和订单。
- 告警处理状态。
- 告警处理记录。
- 严重告警可转售后报修。

客户移动端不展示告警详情。

### 7.3 租赁订单管理

租赁订单以客户和电池为核心。

核心能力：

- 销售创建订单。
- 订单关联客户。
- 订单配置租期、价格、账单日。
- 合同作为订单附属信息。
- 发起销售租赁合同会签。
- 合同会签通过后订单生效。
- 出库时绑定电池 BT码。
- 支持部分出库和分批出库。
- 查看订单账单、回款、欠款、报修记录。

### 7.4 固定资产管理

固定资产管理负责电池组从入库到出库、维修、报废、买断的全过程。

默认资产状态：

```text
在库
维修
租赁
报废
买断
```

状态支持后台配置，但关键状态流转需由业务动作触发。

核心规则：

- 电池出库前必须是“在库”状态。
- 出库后电池状态变为“租赁”。
- 维修完成后必须重新入库，再出库给客户。
- 报废和买断为重要终态，需保留完整记录。

### 7.5 售后报修

售后报修支持客户报修和内部报修。

客户报修规则：

- 客户通过移动端网页扫码或输入 BT码。
- 系统校验 BT码是否属于该客户有效订单。
- 不属于则不允许提交。
- 报修必须填写故障描述、上传图片、填写地址。

内部报修规则：

- 技术人员或资产人员可在后台提交。
- 用于仓库电池、维修电池或内部检测异常场景。

### 7.6 财务管理

财务管理主要覆盖租赁账单、回款、欠款、押金、采购付款等。

账单规则：

- 按月生成。
- 按整月计算。
- 每个订单可配置账单日。
- 账单生成后由财务确认。
- 财务可手动调整金额，但必须记录调整原因。
- 账单生成后 7 天未结清，自动标记逾期。

客户展示规则：

- 客户管理员可查看每期账单明细和欠款总额。
- 客户员工不可查看账单金额和欠款金额。

### 7.7 OA业务审批

OA 一期只做业务审批，不做人事行政全套。

一期审批范围：

- 样品电池送样审批。
- 采购需求审批。
- 采购合同审批。
- 付款申请。
- 销售租赁合同会签。
- 采购入库申请。
- 返厂维修申请。
- 人为损坏赔偿申请。
- 客户退租审批。
- 客户买断审批。
- 质保期内电池报废审批。
- 废旧电池处置。
- 报销流程。
- 借款申请。

审批流要求：

- 后台可配置审批流程。
- 支持不同审批模板。
- 支持审批节点配置。
- 支持附件上传。
- 支持审批记录留痕。
- 审批通过后可自动触发业务状态变化。

## 11. 风险提醒

| 风险 | 影响 | 建议 |
|------|------|------|
| 8个平台字段不一致 | 数据清洗复杂 | 先拿接口字段做统一数据模型 |
| BT码重复或缺失 | 资产、订单、报修都会串数据 | BT码导入和同步必须做唯一校验 |
| 状态配置过于自由 | 报表和流程混乱 | 状态可配置，但关键状态流转受流程控制 |
| 财务可手动改账单 | 金额风险 | 必须保留调整原因和操作日志 |
| 客户多账号权限 | 金额泄露风险 | 客户管理员和客户员工权限必须分开 |
| 分批出库 | 订单状态复杂 | 订单需汇总多个出库单状态 |
| 自动收货 | 客户争议 | 客户端需明确提示 7 天自动收货规则 |
